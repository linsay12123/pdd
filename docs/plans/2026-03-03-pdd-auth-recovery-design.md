# 拼代代账号注册与找回密码设计

日期：2026-03-03

## 背景

当前线上账号流程有两个核心问题：

1. 注册阶段依赖邮箱确认，用户容易卡在“邮件没收到”这一步。
2. 忘记密码还是占位文案，没有真正可用的找回密码流程。

这两个问题叠在一起后，用户既不能稳定注册，也不能自助找回密码，直接影响转化和售后成本。

## 目标

把账号流程改成更直接、更稳定的版本：

- 注册时不再要求邮箱确认。
- 注册成功后直接登录并进入工作台。
- 忘记密码时才发送邮箱邮件。
- 用户点邮件后回到站内页面完成重置密码。
- 处理掉旧流程留下来的“半注册邮箱”脏数据，确保指定邮箱也能继续使用。

## 已评估的方案

### 方案 1：注册直接成功，忘记密码走邮箱

这是本次采用的方案。

优点：

- 用户路径最短，注册不卡邮箱。
- 不再被“确认邮件收不到”卡住主流程。
- 售后解释最简单。
- 找回密码仍然保留邮箱验证，安全性足够。

缺点：

- 注册阶段少了一层邮箱验证。

结论：

对拼代代当前业务阶段最合适，优先保证注册、登录、找回密码稳定可用。

### 方案 2：保留注册邮箱确认

优点：

- 流程更标准。

缺点：

- 当前已经证明邮件链路不稳定。
- 会持续卡住新用户。
- 会继续放大客服负担。

结论：

当前不采用。

### 方案 3：注册先试用，敏感功能再要求邮箱确认

优点：

- 体验比强制确认好。

缺点：

- 逻辑复杂。
- 页面提示和后台状态判断都会明显变重。

结论：

当前不采用，属于过度设计。

## 最终设计

## 1. 注册流程

- 注册页保留现有视觉设计，不改风格。
- 用户填写用户名、邮箱、密码、确认密码后点击“注册账号”。
- 系统调用 Supabase 注册接口。
- 注册成功后直接登录。
- 成功登录后直接进入 `/workspace`。

如果邮箱已经存在：

- 页面提示“这个邮箱已经注册过了，请直接登录，或点击忘记密码重设密码。”

不再保留：

- 注册成功后去邮箱确认
- 重新发送确认邮件
- 注册确认邮件相关提示

## 2. 登录流程

- 继续使用邮箱 + 密码登录。
- 保留当前视觉设计。
- “忘记密码功能稍后补上”替换成可点击的“忘记密码？”入口。

## 3. 忘记密码流程

新增单独的找回密码页面：

- 输入邮箱
- 点击“发送重置密码邮件”

页面提示统一写成：

- “如果这个邮箱已经注册，我们会把重置密码邮件发到你的邮箱，请查看收件箱和垃圾箱。”

这样做的目的：

- 不暴露某个邮箱是否已经注册。

## 4. 重置密码流程

新增站内重置密码页面：

- 用户从邮件点回网站
- 页面要求输入新密码和确认密码
- 保存成功后提示“密码已经修改成功，请重新登录。”
- 然后跳到 `/login`

如果链接无效或过期：

- 页面提示“这个重置链接无效或已过期，请重新发送重置密码邮件。”

## 5. 邮件回跳

忘记密码邮件必须统一回到拼代代站内地址，不允许跳到 `localhost`。

固定规则：

- Site URL 使用正式站点
- 重置密码链接回到站内专门的重置密码路由
- 注册阶段不再依赖邮件，因此注册邮件回跳逻辑整体下线

## 6. 旧邮箱脏数据处理

像 `1318823634@qq.com` 这种被旧流程卡住的邮箱，需要补一层后台清理策略：

- 先查 Supabase 用户状态
- 如果是旧注册流程留下的未完成记录，则删除或重建
- 清理后按新流程重新注册

目标：

- 用户指定邮箱也能继续使用
- 不要求用户换邮箱

## 7. 页面文案和交互

注册页：

- 删除“去邮箱确认”
- 删除“重新发送确认邮件”
- 保留注册成功、已注册、网络失败三种清晰提示

登录页：

- 把忘记密码入口变成可点击按钮

找回密码页：

- 只保留邮箱输入 + 发送按钮

重置密码页：

- 两次密码输入
- 成功提示
- 失败提示

所有按钮要求：

- 点击后有加载状态
- 请求期间禁用，避免连点
- 错误提示紧贴表单，用户一眼能看见

## 8. 错误处理

注册：

- 已注册：提示去登录或忘记密码
- 网络失败：提示稍后再试

忘记密码：

- 不提示“是否注册过”，统一返回模糊成功文案

重置密码：

- 链接失效
- 密码不一致
- 密码长度不足
- 保存失败

都要使用中文直白提示。

## 9. 测试范围

必须覆盖：

- 新用户注册后直接进入工作台
- 已注册邮箱再次注册时出现正确提示
- 忘记密码邮件发送提示正确
- 重置密码链接回站内
- 重置密码成功后可以用新密码登录
- 旧半注册邮箱清理后可重新注册

## 10. 非目标

本次不做：

- 手机号找回
- 第三方登录
- 多因素认证
- 注册时邮箱验证

先把最核心的邮箱登录/找回密码链路做扎实。
