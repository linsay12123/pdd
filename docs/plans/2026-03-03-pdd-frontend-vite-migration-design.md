# 拼代代PDD 前端完整迁移设计（Vite 风格并入 Next 主站）

## 目标

把 `/Users/jeffo/Desktop/自动写作/拼代代-pdd` 里的高审美前端风格，完整迁移到当前线上主项目（Next.js），并且不破坏现有后端能力（任务流程、激活码、下载、接口）。

## 迁移范围

- 首页 `/`
- 登录 `/login`
- 注册 `/register`
- 工作台 `/workspace`
- 我的任务 `/tasks`
- 额度中心 `/billing`（别名 `/recharge`）
- 主应用壳层 `app/(app)/layout.tsx`

## 固定约束

- 品牌统一：`拼代代PDD`
- 图片统一使用主项目 `public` 下素材：
  - `/logo.jpg`
  - `/qrcode.jpg`
- 收费口径不变：
  - 生成文章固定扣 `500` 积分
  - 自动降AI固定扣 `500` 积分
- 不恢复任何在线支付入口（Stripe/支付宝/微信/USDC）

## 技术策略

1. 不换框架，不新建第二站点。  
   继续使用当前 Next 主项目，避免影响已联通的 API 与数据库。

2. 视觉迁移优先。  
   先把页面结构、配色、字体、卡片、按钮迁到主项目。

3. 功能组件复用。  
   工作台与额度中心继续复用现有业务组件（上传、特殊要求、大纲、交付、激活码兑换等），只重做外壳与布局。

4. 边改边验。  
   每批改动后执行 `pnpm lint` + `pnpm test`，保证不引入回归。

## 交付标准

- 用户访问线上域名时，看到的是拼代代设计风格，不再是旧视觉壳。
- Logo 与客服二维码在首页、额度中心、降AI后人工入口等关键位置可见。
- 所有既有测试通过，且核心文案（激活码、500 积分、联系客服）仍可被检索到。
