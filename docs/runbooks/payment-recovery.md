# 额度异常处理手册（激活码模式）

## 场景 1：用户说“激活码无效”

- 先在后台 `/admin` 搜索该激活码。
- 如果查不到，说明码不存在：让用户核对大小写和空格。
- 如果状态是“已使用”，让用户提供账号，核对是不是本人已兑换过。
- 如果状态是“未使用”但用户还失败，记录具体报错文本，再人工复测一次。

## 场景 2：用户说“兑换成功但积分没增加”

- 先让用户刷新页面后再看一次余额。
- 后台核对该激活码状态是否已变成“已使用”。
- 如果后台显示已使用，但用户余额没变，检查额度账本（ledger）是否有这笔入账。
- 如果账本有记录，优先判断为前端未刷新，不要重复补额度。

## 场景 3：用户买了额度但还没收到激活码

- 这是人工销售流程，不是系统自动收款。
- 先确认销售是否已登记收款，然后由管理员批量生成对应档位激活码。
- 发码前先核对用户账号，避免发错人。

## 场景 4：误操作（把激活码发错用户）

- 已经被兑换的码不能二次使用，不要尝试“复活同一激活码”。
- 正确做法是重新生成新码发给正确用户。
- 如果要补偿，直接给用户额外发一个同档位激活码，并在内部登记原因。
