# 支付异常处理手册

## Stripe 没到账

- 先到 Stripe 后台确认订单是否真的支付成功。
- 再检查站点的 Stripe Webhook 日志，看有没有签名失败。
- 如果 Stripe 已支付，但站点没加额度，用同一个 `providerPaymentId` 再补一次回调，系统只会入账一次，不会重复加额度。

## 支付宝或微信没到账

- 先确认商户后台里这笔单是否成功。
- 检查回调地址是否可访问、有没有被拦截。
- 如果平台显示成功，但站点没有入账，重新推送同一笔通知；系统会按订单状态防止重复入账。

## 稳定币人工确认

- 用户走的是人工确认，不是链上自动到账。
- 你先去看自己的钱包地址是否真的收到了款。
- 确认后，再在后台给这个用户手动充值额度。
- 没看到款，就不要充值。

## 反复重试前先看什么

- 订单号是否正确。
- 支付渠道是否对应。
- 用户账号是否正确。
- 这笔订单是否已经显示为 `paid`。

## 最后的兜底

- 如果支付平台成功、系统也已经显示 `paid`，但用户说余额没变，就检查额度账本记录。
- 如果账本里已经有入账记录，优先判断是不是前端页面没刷新，而不是立刻再补一次额度。
