# 支付账号开通手册

这份手册只做一件事：把你需要亲自去注册的支付账号，按顺序拆开。

原则很简单：

- 能由我在代码里先准备的，我先准备
- 必须由你本人去实名、上传证件、绑定银行卡的，我会停在那一步等你
- 我们一次只办一个，避免你同时开太多后台把自己搞乱

## 推荐顺序

1. `Stripe`
2. `支付宝`
3. `微信支付`

原因：

- `Stripe` 最适合先打通海外卡收款
- `支付宝` 和 `微信支付` 都涉及国内商户资料，通常更花时间
- 稳定币这条你已经决定走人工确认，所以不需要先注册额外支付平台

## Stripe

你要做的事：

1. 注册 Stripe 商家账号
2. 完成身份和主体审核
3. 开启正式收款（也就是 live mode，不是测试模式）
4. 后面把这两个值给我：
   - `STRIPE_SECRET_KEY`
   - `STRIPE_WEBHOOK_SECRET`

你通常要准备：

- 常用邮箱
- 手机号
- 你的姓名
- 银行账户
- 如果 Stripe 要求主体资料，还需要公司资料或个体经营资料

我这边已经准备好的：

- Stripe 充值下单接口
- Stripe Webhook 回调接口
- 订阅同步和每月发额度逻辑

你注册完以后，把密钥发给我，我继续帮你接正式收款。

## 支付宝

你要做的事：

1. 注册支付宝开放平台商家账号
2. 完成商户认证
3. 开通你要用的支付能力
4. 后面把商户参数给我

通常要准备：

- 主体证件
- 收款银行卡
- 联系人信息
- 商户资料

我这边已经准备好的：

- 支付宝下单接口骨架
- 支付宝异步通知入口

等你开通好，我再帮你把真实参数接进去。

## 微信支付

你要做的事：

1. 注册微信支付商户平台账号
2. 完成商户审核
3. 开通对应支付产品
4. 后面把商户参数给我

通常要准备：

- 主体证件
- 结算银行卡
- 联系人信息
- 商户资料

我这边已经准备好的：

- 微信下单接口骨架
- 微信支付回调入口

等你开通好，我再把真实参数接进去。

## 稳定币（你这条不需要注册支付平台）

你已经定好的方案是：

- 收 `USDT`
- 收 `USDC`
- 支持 `Base / Ethereum / Solana`
- 客户付款后，你人工确认到账
- 你确认后，再给客户手动充值

所以你只需要后面把 6 个收款地址给我，不需要额外开通第三方支付商户。
