# 任务失败处理手册

## 什么时候可以手动重试

- 任务状态必须已经是 `failed`。
- 只能从安全步骤重新开始，比如重新写正文、重新校字数、重新核验引用、重新导出。
- 手动重试不会再次扣额度，避免重复收费。

## 标准处理流程

1. 先看失败发生在哪一步。
2. 先修外部原因，比如密钥失效、支付回调错误、第三方接口超时。
3. 再用管理员重试工具，把任务从合适的步骤重新拉起。
4. 确认任务状态已经从 `failed` 变成新的执行步骤。

## 不要这样做

- 不要在任务还不是 `failed` 的时候强行重试。
- 不要把任务直接改成完成状态。
- 不要为了省事再扣一次额度。

## 重试后要检查什么

- 任务状态是否已经推进。
- 监控日志里是否记录了这次重试。
- 重试次数有没有持续增加，避免死循环。
- 用户最终是否拿到了 Word、PDF 和降 AI 入口。
